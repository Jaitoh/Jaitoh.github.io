[Coding] Git and Github

Posted on 2020-04-15 Edited on 2021-07-23 In coding Views:

Git ang Github basic commands 安装与使用

git 安装与配置

安装git
sudo apt-get install git
设置用户名和邮箱
git config --global user.name "YOUR NAME"
git config --global user.email "YOUR EMAIL ADDRESS"

git 指令

指令 作用
add 添加文件内容至索引
bisect 通过二分查找定位引入 bug 的变更
branch 列出、创建或删除分支
checkout 检出一个分支或路径到工作区
clone 克隆一个版本库到一个新目录
commit 记录变更到版本库
diff 显示提交之间、提交和工作区之间等的差异
fetch 从另外一个版本库下载对象和引用
grep 输出和模式匹配的行
init 创建一个空的 Git 版本库或重新初始化一个已存在的版本库
log 显示提交日志
merge 合并两个或更多开发历史
mv 移动或重命名一个文件、目录或符号链接
pull 获取并整合另外的版本库或一个本地分支
push 更新远程引用和相关的对象
rebase 本地提交转移至更新后的上游分支中
reset 重置当前HEAD到指定状态
rm 从工作区和索引中删除文件
show 显示各种类型的对象
status 显示工作区状态
tag 创建、列出、删除或校验一个GPG签名的 tag 对象

创建本地版本库

  1. 新建文件夹 创建新的 repository
    mkdir learngit
    cd learngit

  2. repository 初始化
    git init

  3. 文件夹添加文件
    git add filename.cpp

  4. 提交到 repository (-m “提交说明/描述”)
    git commit -m "add a new file"

添加远程仓库(github连接)

A. SSH 加密传输

  1. 创建SSH key
    ssh-keygen -t rsa -C "myemail@example.com"
    usr/.ssh - id_rsa(私钥) id_rsa.pub(公钥)

  2. Github 登录 - Account settings - SSH Keys 粘贴 id_rsa.pub

  3. 关联本地仓库 (origin 代表远程库，git 默认叫法)
    git remote add origin git@github.com:jaitoh/learngit.git

  4. 第一次推送本地库内容到github -u
    git push -u origin master

  5. 修改内容再次推送到github仓库:
    git push origin master

B. HTTPS 连接
不需要SSH key, 其余操作相同， 3.关联仓库命令不同：
git remote add origin https://github.com/jaitoh/learngit.git

从github克隆

  1. github 上创建新库 learngit, 勾选README选项

  2. 克隆到本地

SSH 连接： git clone git@github.com:jaitoh/learngit.git

HTTPS连接:git clone https://github.com/jaitoh/learngit.git

  1. 本地修改代码， 推送到github库:
    git push origin master
git github
